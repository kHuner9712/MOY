# MOY v1.1 销售日常作战系统 - P1 完成版

> **当前版本：v1.1 P1** | 基于 Phase 16 (v1.0) 的增量升级

---

## 1. v1.1 目标

**让一线销售每天离不开 MOY**

核心原则：
- 先服务销售，再服务管理
- 能自动抓的，不要求销售重复录入
- 每次沟通都必须落到"下一步动作"

---

## 2. 已落地的用户入口

### 路径 1：今日作战台
**入口**：`/today` 页面顶部

**解决什么问题**：销售打开首页就能看到今天最该联系的客户，不漏节奏

**展示内容**：
- 今日必须联系客户
- 节奏中断（超过 24h 未推进）
- 高意向沉默（超过 48h 未推进）
- 报价等待回复
- 待发资料 / 等待确认

**每个队列项展示**：
- 客户名
- 风险/原因标签
- 上次动作时间
- 推荐下一步
- 跳转客户详情入口

**已集成文件**：
- `app/(app)/today/page.tsx` - 页面集成
- `components/sales-desk/sales-desk-queue.tsx` - UI 组件

---

### 路径 2：一次沟通，五处复用
**入口**：`/capture` 页面

**解决什么问题**：一次录入，自动生成多种可用内容，减少重复劳动

**展示内容**（确认落库后）：
- ✨ 正在生成五处复用内容...（loading 状态）
- 下一步动作建议
- 给经理汇报（一句话）
- 给客户消息草稿
- 规则生成标记（如 AI 不可用时）

**已集成文件**：
- `app/(app)/capture/page.tsx` - 五处复用结果展示

---

### 路径 3：客户统一时间线
**入口**：`/customers/[id]` 页面

**解决什么问题**：在一个页面看到客户的完整历史，不碎片化

**展示内容**：
- 按时间倒序整合：沟通、跟进、触点、结果、事件、简报
- 不同类型有不同图标和颜色
- AI 生成的内容有标记
- 优先级边框（高/中/低）

**已集成文件**：
- `app/(app)/customers/[id]/page.tsx` - 替换原 FollowupTimeline
- `components/customers/customer-unified-timeline.tsx` - 统一时间线组件

---

### 路径 4：会前 Prep / 会后收口
**入口**：`/customers/[id]` 页面（已有）

**解决什么问题**：会议前快速了解客户，会后快速收口

**已有能力**：
- 一键生成 Meeting Prep 卡片
- 支持 followup / quote / meeting 类型
- 包含关键问题、风险标记、机会信号

## 4. 三条用户路径验证

### 路径 1：今日作战台 → 客户详情
```
/today → 看到作战台队列 → 点击客户 → 进入客户详情 → 查看统一时间线
```

### 路径 2：捕获沟通 → 五处复用
```
/capture → 录入/确认一次沟通 → 页面上直接看到五处复用结果
```

### 路径 3：客户详情 → Prep
```
/customers/[id] → 生成 Prep / 查看时间线 → 能理解当前客户状态
```

| 方面 | v1.0 | v1.1 P2 |

## 6. P2 体验增强（已完成）

### A. 五处复用：一键复制 / 一键采用
- **复制**：客户消息草稿、经理汇报均可一键复制到剪贴板
- **采用**：Next Step 可标记"已采用"
- **状态反馈**：按钮文案变化（复制 → 已复制 ✓）

### B. 作战台排序与筛选
- **筛选**：按类型筛选（全部 / 节奏中断 / 高意向沉默 / 报价等待）
- **排序**：按优先级 / 按最近动作时间
- **上限显示**：最多展示 10 项，超出提示

### C. 时间线类型筛选
- **按钮式筛选**：全部 / 跟进 / 触点 / 事件
- **默认展示全部**
- **自动检测可用类型**

### D. 交互打磨
- 按钮文案更贴近销售场景
- 空状态清晰说明
- 筛选状态有反馈
